## 五、工程师建议

### 1. 先把 v0.15.0 + v0.16.0 提交了

两个版本的代码都在本地，没有 commit。建议：
- v0.15.0 单独一个 commit（小组件 + MCP + 安全 + DB导出 + 生物锁）
- v0.16.0 单独一个 commit（拖拽 + 闹钟 + UI 打磨）
- 各自打 tag

不要继续堆新功能。现有代码需要先固化。

### 2. 开源合规是第一优先级

没有 LICENSE 文件 = 别人不能用你的代码。NOTICE 是法律要求。这两件事加起来 10 分钟就能做完，应该立刻做。

### 3. 数据库迁移要在有用户之前做好

现在 DB schema 是 v6，每次改表都是直接改 schemaVersion + 重建。一旦有真实用户，改表就必须走 migration。建议在开源前加上 Drift 的 migration 策略。

### 4. 做一次真机端到端测试

在模拟器/桌面上测和真机差异很大。建议：
- Android 真机：装上 → 创建事件 → 设置 CalDAV → 同步 → 收到通知
- iOS 真机：同上 + Face ID 解锁 + 小组件

### 5. CI/CD 先修好再开源

CI 挂着就开源，别人 PR 也没法验证。修复只需改两行 yml，应该立刻做。

### 6. 下一版本建议只做 P0

v0.17.0 范围建议：LICENSE + NOTICE + CI 修复 + API Key 掩码 + DB 迁移 + commit 归档。全部是工程基础设施，不涉及新 UI，风险低。
